# План для агента-кодера: оптимизация draggable-карточек библиотеки средств Fortis

## Контекст

В библиотеке средств защиты карточки сейчас слишком большие и перегруженные. При этом важно сохранить ключевую механику: карточки являются draggable-объектами, которые пользователь перетаскивает на карту/эшелон для размещения.

Нужно оптимизировать UX/UI карточек так, чтобы библиотека стала компактнее, быстрее сканировалась и лучше поддерживала сценарий “взять средство и поставить на карту”.

---

# Главная цель

Переделать карточки средств защиты в библиотеке из больших информационных карточек в компактные draggable-карточки/юниты.

Целевая логика:

```txt
Выбрал эшелон → увидел библиотеку средств → взял карточку мышкой → перетащил на карту → отпустил внутри эшелона → объект создан
```

Карточка должна отвечать на 4 вопроса:

1. Что это за средство?
2. Можно ли его поставить на карту?
3. Какое покрытие оно даёт?
4. Сколько уже размещено/добавлено?

Всё остальное должно уходить в детальную панель по клику.

# 2. Разделить типы карточек

## Нужно поддержать 3 типа средств

### 2.1. Draggable map-object

Для средств, которые ставятся точкой на карту.

Примеры:

- РЛС;
- ОЭП;
- тепловизор;
- акустический массив;
- РЭБ;
- турель;
- мобильная группа.

UX:

```txt
Карточка draggable.
Можно перетащить на карту.
После drop создаётся PlacedDefenseObject.
```

### 2.2. Add-only non-physical asset

Для средств, которые не размещаются физически на карте.

Примеры:

- военная комендатура;
- региональный центр МЧС;
- OSINT-мониторинг;
- штаб;
- ПО;
- внешний сервис.

UX:

```txt
Карточка НЕ draggable.
Вместо “Разместить” использовать “Добавить” / “Включить”.
Не показывать preview покрытия на карте.
```

### 2.3. Draw-zone object

Для средств, которые размещаются линией/зоной/полигоном.

Примеры:

- инженерные заграждения;
- сетки;
- тросы;
- защитные конструкции;
- периметр.

UX:

```txt
Карточка draggable или имеет кнопку “Нарисовать”.
После drop/выбора включается режим рисования линии или полигона.
```

# 3. Новая структура компактной карточки

## 3.1. Для draggable map-object

Высота карточки: ориентир 88–112 px.

```txt
┌─────────────────────────────────────────────┐
│ ⋮⋮  [icon 48–56]  Мобильный РЛС       3/6   │
│                  Обнаружение · круг 75 км   │
│                  20 млн ₽ · перетащите      │
└─────────────────────────────────────────────┘
```

Обязательные элементы:

- drag handle;
- иконка или маленькая миниатюра;
- название;
- категория;
- тип покрытия;
- радиус/дальность, если есть;
- цена, если есть;
- счётчик размещения;
- hover-подсказка “Перетащите на карту”.

## 3.2. Для non-physical asset

```txt
┌─────────────────────────────────────────────┐
│ [icon 48–56]  Военная комендатура    11 ед. │
│             Раннее предупреждение · без карты│
│             12 млн ₽ / шт        [+ Добавить]│
└─────────────────────────────────────────────┘
```

Правила:

- не показывать drag handle;
- не использовать слово “Разместить”;
- использовать “Добавить” или “Включить”;
- текст “не требует размещения” заменить на короткий бейдж “Без карты”;
- “Размещено” заменить на “Включено”, если объект не физический.

## 3.3. Для zone-object

```txt
┌─────────────────────────────────────────────┐
│ ⋮⋮  [icon 48–56]  Защитная сетка       2 уч. │
│                  Инженерная защита · линия  │
│                  8 млн ₽ · нарисовать       │
└─────────────────────────────────────────────┘
```

Правила:

- показывать тип геометрии: линия / полигон / зона;
- после drag включать режим рисования;
- не создавать точечный объект без геометрии.

---

# 4. Что убрать из текущей карточки

Убрать из компактного состояния:

- большую картинку на всю ширину;
- крупный бейдж `РАЗМЕЩЕНО`;
- дублирование `Размещено: N`;
- длинное описание в несколько строк;
- большую кнопку `−`;
- большую синюю кнопку на половину карточки;
- пользовательский термин `слот`;
- технические подписи без пользы для пользователя.

Большая картинка, подробное описание, экспертные параметры и расширенные настройки должны быть в drawer/panel деталей.

# 5. Drag-and-drop UX

## 5.1. До drag

Карточка выглядит компактно, но явно показывает, что её можно взять:

- drag handle `⋮⋮`;
- cursor: `grab`;
- hover-состояние;
- подсказка “Перетащите на карту”.

## 5.2. Hover

При наведении на draggable-карточку:

- слегка подсветить карточку;
- показать микро-подсказку;
- можно подсветить активный эшелон на карте.

Пример подсказки:

```txt
Перетащите на карту внутри выбранного эшелона
```

## 5.3. Drag started

Не тащить всю большую карточку.

Создать компактный drag-preview:

```txt
[icon] Мобильный РЛС
круг 75 км
```

На карте одновременно показывать preview покрытия:

- круг;
- сектор;
- линия;
- полигон;
- или ничего, если coverageType = none.

## 5.4. Drag over valid zone

Если курсор над допустимой зоной:

```txt
Можно разместить в L2
```

Визуально:

- подсветить эшелон зелёным;
- preview покрытия сделать нормальным/позитивным;
- курсор drop-allowed.

## 5.5. Drag over invalid zone

Если курсор вне активного эшелона или средство несовместимо:

```txt
Нельзя разместить вне L2
```

Визуально:

- подсветить зону красным;
- preview покрытия сделать warning/error;
- drop не должен создавать объект, если это hard error.

## 5.6. Drop

После drop:

1. Проверить геометрию.
2. Проверить совместимость.
3. Создать `PlacedDefenseObject`.
4. Показать объект на карте.
5. Показать покрытие объекта.
6. Обновить счётчик в карточке.
7. Добавить объект в панель объектов эшелона.
8. Пересчитать калькулятор/summary.

---

# 6. Детальная панель средства

## Цель

Не перегружать карточку деталями, но сохранить доступ к информации.

## Поведение

При клике по карточке открыть drawer/panel с деталями средства.

В деталях показывать:

- большое изображение/3D-превью;
- полное название;
- описание;
- категория;
- роль;
- тип размещения;
- тип покрытия;
- радиус/угол/дальность;
- цена;
- совместимость с эшелонами;
- сколько размещено;
- список размещённых объектов этого типа;
- кнопки:
  - `Разместить на карте`;
  - `Добавить`;
  - `Настроить`;
  - `Показать размещённые`.

# 7. Состояния карточки

## 7.1. Не размещено

```txt
На карте: 0/6
```

## 7.2. Частично размещено

```txt
На карте: 3/6
```

## 7.3. Лимит достигнут

```txt
Лимит: 6/6
```

Кнопку/drag disabled, если лимит запрещает новые объекты.

## 7.4. Невозможно разместить в активном эшелоне

```txt
Не подходит для L2
```

Карточка остаётся видимой, но muted/disabled или с warning.

## 7.5. Non-physical asset включён

```txt
Включено: 11 ед.
```

# 8. Терминология

## Запрещено в UI

Не использовать:

```txt
slot
слот
РАЗМЕЩЕНО
не требует размещения
```

## Использовать

Для физических средств:

```txt
На карте: 3/6
Разместить
Перетащите
Покрытие
```

Для нефизических средств:

```txt
Включено: 11
Добавить
Включить
Без карты
Без покрытия
```

Для зональных средств:

```txt
Участков: 2
Нарисовать
Линия
Полигон
Зона
```

# 9. Layout библиотеки

## Если библиотека сбоку

Использовать горизонтальные карточки:

```txt
┌────────────────────────────────────┐
│ ⋮⋮  🛰  Мобильный РЛС        3/6    │
│      Обнаружение · круг 75 км       │
│      20 млн ₽ · перетащите          │
└────────────────────────────────────┘
```

## Если библиотека снизу

Использовать компактные плитки:

```txt
┌──────────────┐
│ ⋮⋮           │
│     🛰        │
│ РЛС          │
│ круг 75 км   │
│ 3/6 · 20 млн │
└──────────────┘
```

Для текущего интерфейса предпочтительно сделать вертикальный список горизонтальных карточек высотой 88–112 px.

---

# 10. Связь с картой

## Нужно проверить

1. Карточка физического средства draggable.
2. При drag активный эшелон подсвечивается.
3. При drop внутри эшелона объект создаётся.
4. При drop вне эшелона объект не создаётся.
5. Счётчик карточки обновляется.
6. Объект появляется в панели объектов.
7. Калькулятор пересчитывается.
8. Покрытие объекта отображается, если уже реализован coverage layer.

---

# 11. Связь с объектной панелью

Удаление и уменьшение количества не должны быть основным действием в библиотеке.

Правильная логика:

- библиотека отвечает за добавление/размещение;
- объектная панель отвечает за управление уже размещёнными объектами.

Поэтому:

- убрать большую кнопку `−` из карточки библиотеки;
- удалить/перенести/изменить quantity можно в карточке размещённого объекта;
- если нужен быстрый `+`, оставить компактную кнопку, но не делать её главным элементом.

# 12. Acceptance criteria

## Проверка 1: карточка стала компактной

1. Открыть библиотеку средств.
2. Сравнить высоту карточки.

Ожидаемый результат:

- карточка занимает примерно 88–112 px по высоте;
- на экране помещается больше средств;
- большая картинка не занимает половину карточки.

## Проверка 2: physical asset draggable

1. Выбрать физическое средство, например РЛС.
2. Навести на карточку.
3. Перетащить на карту.

Ожидаемый результат:

- карточка показывает drag affordance;
- появляется drag-preview;
- активный эшелон подсвечивается;
- drop внутри эшелона создаёт объект.

## Проверка 3: non-physical asset не draggable

1. Найти средство типа “Военная комендатура”.
2. Проверить карточку.

Ожидаемый результат:

- нет drag handle;
- нет текста “Разместить”;
- есть кнопка “Добавить” или “Включить”;
- drag на карту не запускается.

## Проверка 4: zone-object

1. Выбрать средство типа линия/полигон.
2. Начать размещение.

Ожидаемый результат:

- интерфейс не создаёт точечный объект сразу;
- включается режим рисования линии/полигона;
- карточка показывает тип размещения.

## Проверка 5: счётчики

1. Разместить объект на карте.
2. Вернуться к карточке средства.

Ожидаемый результат:

- счётчик изменился, например `На карте: 1/6`;
- для non-physical asset счётчик выглядит как `Включено: 1`.

## Проверка 6: нет старых терминов

Ожидаемый результат:

- в UI нет `slot`;
- в UI нет `слот`;
- в UI нет крупного бейджа `РАЗМЕЩЕНО` на карточке библиотеки.

## Проверка 7: связь с калькулятором

1. Перетащить средство на карту.
2. Открыть калькулятор.

Ожидаемый результат:

- объект попал в расчёт;
- количество и стоимость обновились;
- объект относится к выбранному эшелону.

---

# 13. Не менять без необходимости

Не переписывать всю карту и store с нуля.

Нужно точечно изменить:

- UI карточки средства;
- drag preview;
- правила draggable / non-draggable;
- тексты;
- счётчики;
- связь с object placement, если она уже есть.

Визуальный стиль приложения сохранить: тёмная карта, зелёные/синие акценты, карточная структура, текущая сетка и отступы.

```txt
Найди текущий компонент карточки средства в библиотеке и оптимизируй его под draggable UX. Карточка должна стать компактной, высотой около 88–112 px, с drag handle только для физических объектов, без большой картинки, без user-facing slot/слот, без крупного бейджа РАЗМЕЩЕНО. Раздели поведение на map-object, non-physical asset и zone-object. Для map-object оставь drag-and-drop на карту с компактным drag-preview и обновлением счётчиков. Удаление и уменьшение количества перенеси в объектную панель, а библиотеку оставь местом добавления/размещения. Проверь acceptance criteria из этого файла.
```
