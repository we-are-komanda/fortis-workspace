# Handoff: Fortis — лендинги (вариант 1 и вариант 2)

## Обзор
В пакете — два **готовых дизайна лендинга** продукта Fortis (модульная финансово-аналитическая ГИС-платформа планирования защиты объектов). Оба несут одни и те же ключевые тезисы: **модульность** (легко дорабатывать и подключать модули), **современная команда и зрелые процессы**, **полное соответствие ТЗ**, плюс сквозной нарратив **«проблема → решение»**.

- **`v1/`** — тёмный «mission-control» вариант (доработка исходного лендинга). Кинематографичная аналитическая эстетика, есть переключатель светлой/тёмной темы.
- **`v2/`** — светлый продуктовый вариант (новое направление). Fintech/BI-регистр, бенто из модулей, диаграмма «ядро + модули». Тоже с переключателем светлой/тёмной темы.

Каждый вариант самодостаточен: откройте `index.html` в браузере — работает без сборки и сервера.

## О файлах дизайна
Файлы в пакете — это **дизайн-референсы, сделанные на HTML/CSS/JS**: прототипы, показывающие задуманный внешний вид и поведение. Это **не production-код для прямого копирования**. Задача — **воссоздать эти дизайны в вашем рабочем окружении** (React / Vue / Next.js и т.д.) по принятым в проекте паттернам и библиотекам. Если окружения ещё нет — выбрать подходящий фреймворк и реализовать дизайн в нём.

Вёрстка намеренно «канонична» (явные закрывающие теги, двойные кавычки) и легко переносится в компоненты.

## Фиделити
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, состояния и анимации. UI воссоздавать пиксель-в-пиксель, используя библиотеки и паттерны вашего кодовой базы. Цвета заданы в **OKLCH** через CSS-переменные — переносите токенами, а не хардкодом.

## Структура каждого варианта

| Файл | Назначение |
|---|---|
| `index.html` | Разметка всех секций |
| `styles.css` | База: токены (`:root`), типографика, секции, тёмная тема, адаптив |
| `script.js` | Sticky-навигация, счётчики, reveal по скроллу, переключатель темы (`localStorage`) |
| `flow.css` | Общий модуль секций «Проблема → Решение». Использует токены `--fl-*`, которые в конце `styles.css` маппятся на токены варианта |
| `flow.js` | Анимации/скролл-ревилы блока «Проблема → Решение» (IntersectionObserver + демо-циклы) |
| `screens/*.png` | Реальные скриншоты продукта (карта, эшелоны, калькулятор) |

> **Важно про темизацию.** Тема переключается атрибутом `data-theme="light|dark"` на `<html>`. Анти-фликер-скрипт в `<head>` выставляет тему до отрисовки (из `localStorage` ключ `fortis-v2-theme`, иначе по `prefers-color-scheme`). `flow.css` не знает про конкретный вариант — он читает `--fl-*`, а маппинг этих переменных на палитру варианта лежит в конце `styles.css`. При переносе сохраните эту развязку: один общий модуль секций + тонкий слой маппинга токенов на каждый бренд/тему.

## Секции (одинаковый порядок в обоих вариантах)

1. **Hero** — заголовок, лид, CTA, «леджер» из 3 метрик (счётчики), макет продукта (живая карта с эшелонами + бюджет-панель + плавающая карточка модуля).
2. **Проблема — «Как это устроено сейчас»** (`#problem`, из `flow.css`):
   - Анимированная «сломанная ручная цепочка» из 4 этапов (Обычная карта → Excel → Формулы → «?») с бейджами трения между ними (бегущий пунктир, пульсация).
   - Сетка из 6 «болей» с иконками.
3. **Решение — «Как решает Fortis»** (`#solution`, из `flow.css`):
   - Хаб «Единая модель» — карта/калькулятор/отчёт стянуты к ядру светящимися связями.
   - 4 фичи с **работающими демо** (запускаются при попадании в вьюпорт; ряды **выезжают с боков** при скролле): живой пересчёт карты, подбор под бюджет, сравнение A/B, версии + совместная работа.
4. **Дальше — секции конкретного варианта:**
   - **v1:** Эшелоны (диаграмма), Как работает, МОГ (составной объект), Модули, Соответствие ТЗ (матрица), реальные экраны, Команда, CTA.
   - **v2:** Trust-полоса, Бенто модулей, Маршрут пользователя, Модульность («ядро + подключаемые модули»), Соответствие ТЗ (таблица), реальные экраны, Команда + полоса «Признанные мировые технологии», CTA.

## Дизайн-токены

Оба варианта используют **OKLCH** и `color-mix(in oklch, …)`. Точные значения — в `:root` (и блоке `[data-theme="dark"]`) внутри `styles.css` каждого варианта. Ниже — ориентиры; **источник истины — CSS-файлы**.

### v1 — тёмный mission-control (светлая/тёмная темы в `styles.css`)
- Шрифты: **Unbounded** (дисплей), **Manrope** (текст), **JetBrains Mono** (подписи/числа).
- Семантика: `--bg`, `--bg-2`, `--surface`, `--surface-solid`, `--ink/-2/-3`, `--line/-strong`, `--accent`, `--accent-2`, `--accent-ink`, `--ok`, `--warn`, эшелоны `--e1…--e4`.

### v2 — светлый fintech/BI (светлая/тёмная темы в `styles.css`)
- Шрифты: **Space Grotesk** (дисплей), **Manrope** (текст), **JetBrains Mono** (подписи/числа).
- Семантика: `--paper`, `--paper-2`, `--card`, `--ink/-2/-3`, `--line/-2`, `--accent`, `--accent-press`, `--accent-tint`, `--ink-on-acc`, `--ok`, `--warn`, эшелоны `--e1…--e4`.

### Общие (для блока «Проблема → Решение», `--fl-*`)
Не задавайте их напрямую — они маппятся на токены варианта в конце `styles.css` (`--fl-accent`, `--fl-warn`, `--fl-ink*`, `--fl-surface`, `--fl-e1…4`, `--fl-radius*`, `--fl-shadow*`, `--fl-ease`, `--fl-font-display/-mono`).

### Радиусы / тени / easing
- Радиусы: `--radius`, `--radius-sm` (см. CSS; v2 ~20/13px, v1 свои).
- Тени: `--shadow`, `--shadow-sm`, `--shadow-lg`.
- Easing: `--ease: cubic-bezier(0.16, 1, 0.3, 1)` — основной для всех переходов/ревилов.

## Интеракции и поведение

- **Reveal по скроллу:** `IntersectionObserver` добавляет класс `is-in` / `fl-in`. Есть **failsafe**: если обсервер не сработал (некоторые встраивания), контент всё равно показывается — не оставляйте элементы «застрявшими» на `opacity:0`.
- **Счётчики чисел:** анимируются при первом появлении (`data-count`, `data-money`, `data-prefix`, `data-suffix`).
- **Демо в секции «Решение»** (`flow.js`, по `data-fl-demo`):
  - `live` — маркер ходит по карте, цена и бюджет-бар пересчитываются по циклу.
  - `budget` — средства поэтапно отмечаются по приоритету до лимита, появляется «в рамках бюджета».
  - `compare` — счётчики двух вариантов + бейдж «−18%».
  - `collab` — курсоры коллег по карте + таймлайн версий.
  - Демо **останавливаются вне вьюпорта** (производительность) и **полностью отключаются** при `prefers-reduced-motion`.
- **Выезд фич с боков:** в `#solution` текст и его демо въезжают с противоположных краёв, направление чередуется по рядам (`flow.css`, `.fl-feat … translateX`).
- **Переключатель темы:** кнопка в навигации, иконка солнце/луна; выбор сохраняется в `localStorage`; первый визит — по системной теме; без фликера при загрузке.
- **Адаптив:** брейкпоинты ~1000px и ~640px (v2) / аналогично v1. Цепочка «проблемы» на узких экранах разворачивается вертикально; бенто/таблицы/демо переходят в одну колонку.

## State (при переносе в компоненты)
- `theme: 'light' | 'dark'` (персист в `localStorage`).
- Флаги «в зоне видимости» для каждой reveal-секции/демо (через IntersectionObserver или аналог во фреймворке).
- Внутренние индексы циклов демо (`live`/`budget`/`compare`/`collab`) — локальный стейт компонента, чистите таймеры при размонтировании/выходе из вьюпорта.
- `prefers-reduced-motion` — уважать: при включённом отдавать конечное состояние без анимаций.

## Ассеты
- `screens/*.png` — реальные скриншоты продукта (карта с эшелонами, маркеры, калькулятор-смета, подбор под бюджет). Используются в секции «реальные экраны».
- Все иконки — инлайновые SVG в разметке (stroke-иконки), отдельных файлов нет.
- Шрифты подключаются с Google Fonts (`<link>` в `<head>`). В вашем проекте используйте self-host или принятый способ подключения.
- Диаграммы (эшелоны, хаб модулей, демо) нарисованы на CSS/SVG — внешних изображений не требуют.
